# RAG Project

פרויקט זה מממש זרימת RAG (Retrieval-Augmented Generation) על מסמכי תיעוד Markdown.
המטרה היא לאפשר לסוכן לענות על שאילתות טכנולוגיות ותיעודיות באמצעות שילוב של:
- שליפה סמנטית מ־Pinecone/LLM
- שכבת מידע מובנה על הקבצים (`structured_data.json`)
- סינתוז תשובות מאורגנות

## מה הפרויקט עושה

- מנתח ומייצר אינדקס של מסמכי Markdown מתוך `kiro/`
- בונה שכבת נתונים מובנית („Structured Data“) המאפשרת מענה על שאילתות מסוג:
  - רשימות של החלטות טכניות
  - כללים והנחיות
  - אזהרות והתראות
  - שמות מקורות ומסמכים
- מפעיל סוכן Gradio שמקבל שאלה ומחזיר תשובה חכמה בארכיטקטורה מבוססת אירועים

## מבנה הפרויקט

- `agent.py` – נקודת כניסה ל־Gradio ולרצת ה־workflow.
- `workflow.py` – לוגיקת זרימת האירועים, ניתוב בין שליפה סמנטית לנתונים מובנים.
- `structured_data.py` – מנתח את קבצי `kiro/*.md` ויוצר מבנה נתונים מובנה ל־JSON.
- `prepare.py` – בונה את האינדקס ואת `structured_data.json` לפני ההרצה.
- `main.py` – מפעיל את היישום.
- `kiro/` – מסמכי תכנון ותיעוד שעל בסיסם נבנה הידע.

## התקנה

1. צור סביבה וירטואלית:
   ```powershell
   python -m venv .venv
   ```
2. הפעל את הסביבה:
   ```powershell
   .venv\Scripts\Activate.ps1
   ```
3. התקן את התלויות:
   ```powershell
   pip install -r requirements.txt
   ```

## הכנה והרצה

1. הפעל את `prepare.py` כדי לבנות את האינדקס ושכבת הנתונים המובנים:
   ```powershell
   python prepare.py
   ```

2. הפעל את הממשק:
   ```powershell
   python main.py
   ```

3. פתח את הכתובת שמופיעה בטרמינל כדי להשתמש ב־Gradio.

## מה לשאול את הסוכן

הנה דוגמאות לשאלות שה־Agent יכול לענות עליהן:

- "תן לי רשימה של כל ההחלטות הטכניות שהוזכרו במסמכים."
- "איזה כללים או הנחיות לגבי RTL קיימים בתיעוד?"
- "מהן האזהרות הרגישות שצוינו בתיעוד?"
- "באיזה מסמכים יש אזכור של Pinecone או Cohere?"
- "מה המקורות העיקריים לדרישות הפרויקט?"

## הערות נוספות

- `structured_data.json` הוא קובץ נגזר שנוצר על ידי `prepare.py`.
- אם רוצים לשמור את הקובץ תחת גרסה, יש לוודא שהוא מתעדכן בכל הפעלה של `prepare.py`.
- ניתן להרחיב את ה־workflow עם רכיבים נוספים כמו מעקב תאריך, סיווג אוטומטי ושכבת רמת ביטחון בתשובות.
